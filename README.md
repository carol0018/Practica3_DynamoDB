## Reflexión Final: DynamoDB en el Ecosistema NoSQL

**Para tu informe final, incluye una sección de reflexión respondiendo a estas preguntas:**

**Comparativa: ¿Qué diferencias clave notaste entre trabajar con DynamoDB y lo que has visto de MongoDB? (Piensa en la sintaxis de inserción, las consultas, el esquema…).**

DynamoDB es una base de datos clave-valor, mientras que MongoDB es una base de datos documental. La sintaxis de insercción de DynamoDB se basa en búsqueda por claves, y la de MongoDB ofrece más flexibilidad para buscar por cualquier campo.

En DynamoDB no tienes que preocuparte de la infraestructura, mientras que MongoDB requiere que gestiones tú mismo los servidores.

**Ventajas Serverless: ¿Qué beneficios crees que aporta un servicio como DynamoDB a un equipo de desarrollo? ¿Y qué inconvenientes podría tener?**

Una base de datos en la nube es muy útil para trabajar desde cualquier dispositivo. Ofrece más flexibilidad. 

Entre sus inconvenientes podemos encontrar el costo potencial elevado si no se optimiza bien o las limitaciones en consultas complejas en comparación con bases de datos relacionales. 

**Experiencia: ¿Qué te resultó más fácil de esta práctica? ¿Qué fue lo más complicado o confuso?**

Lo más fácil la búsqueda de datos a través de los filtros. Lo más confuso al principio la insercción de datos, pero tampoco llega a ser complicado.
